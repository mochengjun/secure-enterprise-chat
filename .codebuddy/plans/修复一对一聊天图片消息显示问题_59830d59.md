## 用户需求

修复一对一聊天中的图片消息显示问题：发送方可以看到自己发送的图片，但接收方无法看到收到的图片消息。

## 问题分析

1. 服务端 mediaConfig.BaseURL 配置为相对路径 `/api/v1/media`，返回的 media_url 和 thumbnail_url 为相对路径（如 `/api/v1/media/xxx/download`）
2. Flutter 客户端 message_bubble.dart 使用 `UrlHelper.toAbsoluteUrl()` 转换相对路径为完整 URL
3. 发送方能看到图片说明 URL 转换逻辑正常，问题在于接收方收到的 WebSocket 消息中 media_url 字段可能为空或字段名不匹配

## 核心问题点

- 发送方：从 API 响应获取消息，包含正确的 media_url
- 接收方：通过 WebSocket 收到的消息，media_url 字段可能为空或字段名与预期不符（服务端可能使用 `mediaUrl` 而非 `media_url`）

## 技术方案

### 问题定位

1. **message_bubble.dart** 第152-156行：检查 `message.mediaUrl != null`，如果为 null 则显示 `[图片]` 占位符
2. **chat_repository_impl.dart** 第108-109行：解析 WebSocket 消息时使用 `data['media_url']`
3. **潜在问题**：服务端返回的字段名可能是 camelCase (`mediaUrl`) 而非 snake_case (`media_url`)，或字段为空

### 修复策略

1. 修改 chat_repository_impl.dart 的 _parseMessage 方法，增加对多种字段名的兼容性处理
2. 增加对 metadata 字段的备用解析（服务端可能将媒体信息放在 metadata 中）
3. 在 message_bubble.dart 中增加更健壮的 URL 获取逻辑

### 涉及文件

- `secure-enterprise-chat/apps/flutter_app/lib/features/chat/data/repositories/chat_repository_impl.dart`
- `secure-enterprise-chat/apps/flutter_app/lib/features/chat/presentation/widgets/message_bubble.dart`

## 实现细节

### 修复点1: chat_repository_impl.dart - _parseMessage 方法

```
// 增加字段名兼容性处理
mediaUrl: data['media_url'] ?? data['mediaUrl'] ?? data['metadata']?['media_url'],
thumbnailUrl: data['thumbnail_url'] ?? data['thumbnailUrl'] ?? data['metadata']?['thumbnail_url'],
```

### 修复点2: message_bubble.dart - _buildImageContent 方法

```
// 增加更健壮的 URL 获取逻辑，处理空字符串情况
String? imageUrl = message.thumbnailUrl?.isNotEmpty == true 
    ? message.thumbnailUrl 
    : (message.mediaUrl?.isNotEmpty == true ? message.mediaUrl : null);
```
