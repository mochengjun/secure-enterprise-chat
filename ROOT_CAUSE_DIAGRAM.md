## 总结

### 问题本质
不是连接数多,而是数据库配置不当导致在极少连接下也会资源耗尽。

### 核心修复
1. 配置 SQLite 连接池 (MaxOpenConns=10)
2. 设置 busy_timeout=5000ms 避免永久阻塞
3. 使用 WAL 模式支持读写并发
4. 添加数据库连接监控

### 预期效果
- 崩溃率: 多次/天 → 0次/周
- 内存占用: 持续增长 → 稳定 < 500MB
- 查询延迟: 100-5000ms → < 100ms
- 连接数: 无限制 → 稳定 < 10
